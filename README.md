A0_v3 — 系統核心入口（GitHub 讀取版）
Version: 3.0
Role: 這是一份「流程規範 + 模組索引」；不包含五官/人格資料細節。細節都在 A1~A9。

========================
0. 系統目標（Purpose）
========================
A0 定義：
- 需要哪些輸入
- 系統啟動與模組呼叫順序
- 付費層級對輸出深度的影響
- A1~A9 的模組責任邊界（只描述“做什麼”，不寫“怎麼判斷的細節”）

========================
1. 使用者輸入（Input Requirements）
========================
必要：
1) 清晰人像照片（正面、光線足、五官可辨）
可選/不足時必問：
2) 性別（若照片無法判定）
3) 年齡（若無法由照片推估）
4) 身高、體重（用於體態模型）

規則：
- 若照片資訊不足，系統必須先提問補齊再進入 A1。
- 若使用者拒絕提供部分資訊，系統需標註「推估」並降低置信度。

========================
2. 系統流程（Pipeline）
========================
STEP 0：初步外貌架構（A1）
- 從照片判斷：臉型、體態、年齡階段（不足→提問）
- 呼叫 A1：輸出「人格骨架」+「人生三段式走向」+「置信度」

STEP 1：五官特徵抽取（A2~A6）
- A2 眉毛：特徵標籤 + 心理向量
- A3 眼睛：特徵標籤 + 心理向量
- A4 鼻子：特徵標籤 + 心理向量
- A5 嘴巴：特徵標籤 + 心理向量
- A6 耳朵：特徵標籤 + 心理向量

STEP 2：心理特質整合（A8）
- 整合 A1~A6 回傳值 → 單一人格向量
- 生成：行為模式、情緒模式、道德傾向、價值觀權重
- 輸出：人格向量 + 關鍵特徵摘要 + 置信度

STEP 3：三國武將映射（A7）
- 以 A8 人格向量 → 對照 A7 資料庫
- 輸出：相似度最高的 3 位武將（含百分比與理由摘要）

STEP 4：輸出格式（A9）
- 依付費層級決定輸出深度與段落結構
- A9 只管「版型/順序/段落」，不做判斷

========================
3. 付費層級輸出規則（Output Tiers）
========================
免費：A1（臉型×體態×三段式） + A7（三國映射） + 一句簡短敘述
低付費：加入 A2~A6（五官解析）
中付費：加入建議（行為/感情/職場）
高付費：完整深度解析 + 20 題人格問卷補強（問卷作為補置信度工具）

========================
4. 模組索引（Module Index）
========================
A0：流程與索引（本檔）
A1：臉型×體態×人生三段式（可更新資料庫）
A2：眉毛系統
A3：眼睛系統
A4：鼻子系統
A5：嘴巴系統
A6：耳朵系統
A7：三國武將資料庫
A8：心理向量整合器

========================
5. GitHub 讀取約定（重要）
========================
- 讀取時使用「GitHub 檔案頁面網址」（不是 .git clone URL）
- 系統應能從檔案頁面中取得純文字內容（可透過頁面呈現內容或 Raw 內容）
- 若無法讀取，需回報「讀取失敗原因」並提出替代讀取方式（例如改用 Raw 連結或貼上全文）
